# 项目规则 - AI上下文

## 项目基本信息
- **项目名称**: Admin Template
- **架构**: 前后端分离
- **主要模块**: `admin-frontend`、`admin-backend`、`admin-mobile`
- **核心技术栈**: Vue 3、TypeScript、Spring Boot、Java 21、MyBatis Plus、MySQL 8.0

## 目录结构规范
- 配置文件、文档和脚本位于项目根目录或各自模块根目录
- 业务逻辑按功能模块组织，避免跨模块直接依赖

## 技术栈规范
- 严格遵循 `tech-stack.md` 中指定的版本
- 优先使用已有工具和库，避免过多外部依赖
- 新增依赖需经过团队评审

## 代码风格要求
- 遵循 `code-style.md` 中定义的代码规范
- 所有代码必须格式化后提交
- 使用 UTF-8 编码，文件末尾添加换行符

## 命名规范
- 包名: 小写字母，点分隔，如 `com.example.backend`
- 类名: 大驼峰，如 `UserService`
- 方法名/变量名: 小驼峰，如 `getUserById`
- 常量名: 全大写，下划线分隔，如 `MAX_PAGE_SIZE`
- 组件名: 大驼峰，如 `UserList.vue`

## 开发流程
- **分支管理**: `main`(稳定版)、`develop`(开发)、`feature/*`(新功能)、`fix/*`(bug修复)
- **提交规范**: 简体中文，格式 `类型(范围): 描述`，类型包括 `feat`、`fix`、`docs`、`style`、`refactor`、`test`、`chore`
- **代码审查**: 所有代码需至少一次审查才能合并

## 测试规范
- 核心业务逻辑必须编写单元测试，覆盖率不低于 80%
- 前后端接口必须编写集成测试
- 关键业务流程必须编写 E2E 测试
- AI 生成的测试用例需人工验证并补充边缘情况

## AI工具使用规则
- **代码生成**: 必须进行充分测试和审查，符合项目规范，禁止直接用于生产环境
- **文档生成**: 必须人工审核修改，保持与代码同步，关键文档由人工编写
- **测试生成**: 需覆盖核心业务逻辑，人工验证并补充边缘情况
- **技能文档**: `.ai/skills` 目录包含AI可以调用的相关技能定义和规范，如果涉及相关操作，请按照文档中指定的步骤来完成

## 安全规范
- 禁止硬编码敏感信息
- 所有输入必须验证过滤
- 密码使用安全哈希算法存储
- 对外接口使用 HTTPS
- 配置合理的 CORS 策略
- 敏感数据加密存储和传输

## 部署规范
- 环境: 开发、测试、预发布、生产
- 部署必须使用自动化脚本，支持回滚
- 所有服务必须配置监控和日志
- 关键操作记录审计日志
- 错误日志实时告警

## 变更管理
- 需求变更需评审批准，记录在需求管理系统
- 技术栈变更需团队评审，提供迁移计划
- 配置变更使用配置管理系统，记录审计日志

## 规则更新
- 规则定期更新，所有成员必须遵守最新版本
- 更新需团队评审并通知所有成员
